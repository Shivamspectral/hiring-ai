# ✦ TalentLens — RAG-Based Agentic AI Hiring Tool

> Built for the **Alta AI Builder Fellowship** · [Live Demo](https://talentlens-ai-hiring.vercel.app) · [GitHub](https://github.com/Shivamspectral/talentlens)

TalentLens is a production-grade **RAG-based Agentic AI hiring tool** that eliminates slow, biased, manual recruitment. At its core is a **Retrieval-Augmented Generation (RAG) pipeline** — job descriptions and resumes are embedded as vectors into Pinecone, enabling semantic similarity search far beyond simple keyword matching. On top of that sits an **autonomous AI agent** powered by LLaMA 3.3 70B that independently reasons, scores, decides, and acts — shortlisting candidates, rejecting others, generating interview questions, and drafting emails — all without any human input.

---

## 🧠 Why RAG + Agentic AI?

Traditional ATS tools match keywords. TalentLens understands meaning.

- **RAG (Retrieval-Augmented Generation)** — resumes and JDs are converted into semantic vectors using Google Gemini Embeddings and stored in Pinecone. When a candidate applies, the system retrieves the most relevant JD context and feeds it to the LLM for deep, context-aware evaluation — not just keyword matching.
- **Agentic AI** — the hiring agent doesn't just answer questions, it takes actions. It autonomously ranks every candidate, makes shortlist/reject decisions, generates personalized interview questions, and drafts emails — operating end-to-end like a real recruiter.
- **LLaMA 3.3 70B via Groq** — all reasoning, scoring, bias detection, and skill gap analysis runs through one of the most capable open-source LLMs, served at lightning speed via Groq API.

## 🎯 What TalentLens Does

| Step | What Happens |
|---|---|
| 📝 **Post a Job** | Paste your JD → AI instantly extracts required skills, experience, and responsibilities |
| 📄 **Upload Resumes** | Upload up to 4 PDFs, TXTs, or DOCXs at once → AI parses every resume automatically |
| 🧠 **AI Scoring** | Each candidate gets a 0–100 match score across skills, experience, education, and confidence |
| ✦ **Run Pipeline** | AI agent auto-shortlists (≥75), flags for review (40–74), or rejects (<40) every candidate |
| 📬 **Auto Emails** | Shortlisted candidates get interview invites · Rejected candidates get personalized feedback |
| 🎯 **Interview Questions** | Top candidates get 10 custom interview questions generated by AI |
| ⚖️ **Bias Detection** | AI scans your JD for gendered, age-biased, and exclusionary language and rewrites it inclusively |
| 📈 **Skill Gap Analysis** | Every candidate gets a personalized week-by-week learning roadmap for missing skills |

## 🤖 How the AI Works

```
You post a Job Description
        ↓
   LLM extracts: role, required skills, experience, responsibilities
        ↓
   JD embedded as vector → stored in Pinecone
        ↓
You upload Resumes (PDF / TXT / DOCX)
        ↓
   LLM parses resume → name, skills, experience, education, work history
        ↓
   Skill graph matching + RAG semantic retrieval from Pinecone
        ↓
   LLM deep evaluation →
        · Overall score (0–100)
        · Skills score, Experience score, Education score
        · Strengths and weaknesses
        · Matched, partial, and missing skills
        ↓
You click "Start AI Screening"
        ↓
   Score ≥ 75  →  Shortlisted + 10 interview questions generated
   Score 40–74 →  Flagged for manual HR review
   Score < 40  →  Rejected + skill gap learning roadmap generated
```

## 📊 Candidate Intelligence

Every candidate in TalentLens gets a full AI-generated profile:

- **Match Score** — 0 to 100, ranked against the job
- **Shortlisted / Review / Rejected** status assigned automatically
- **Strengths** — what makes this candidate stand out
- **Weaknesses** — where they fall short
- **Skill Gap Report** — for rejected/review candidates:
  - Every missing skill listed with priority (HIGH / MEDIUM / LOW)
  - Estimated weeks to learn each skill
  - Free and paid learning resources with links
  - Personalized learning path summary written by AI
- **Interview Questions** — for shortlisted candidates:
  - 4 technical questions based on matched skills
  - 2 gap questions probing missing areas
  - 2 behavioral questions
  - 2 situational questions

## ⚖️ Built-in Bias Detection

TalentLens scans every job description for:
- **Gendered language** — "rockstar", "ninja", "manpower", "chairman"
- **Age bias** — "digital native", "young and hungry", "recent graduate"
- **Exclusionary phrases** — "no gaps in employment", "must work long hours"
- **Cultural bias** — overly narrow culture-fit language

For each issue, AI provides the original phrase, an inclusive replacement, and the reason for the change. It also generates a fully rewritten inclusive JD.

## ⚙️ Setup & Installation

### Prerequisites
- Python 3.10+
- Node.js 18+
- Accounts on: [Groq](https://console.groq.com), [Supabase](https://supabase.com), [Pinecone](https://pinecone.io), [Google AI Studio](https://aistudio.google.com)

### 1. Clone the repo
```bash
git clone https://github.com/Shivamspectral/talentlens.git
cd talentlens
```

### 2. Backend setup
```bash
cd backend
pip install -r requirements.txt
```

Create `.env` in `/backend`:
```env
GROQ_API_KEY=your_groq_key
GEMINI_API_KEY=your_gemini_key
PINECONE_API_KEY=your_pinecone_key
PINECONE_INDEX=hiring-pipeline
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your_supabase_anon_key
FRONTEND_URL=https://talentlens-ai-hiring.vercel.app
```

Run Supabase SQL setup:
```sql
create table if not exists jobs (
  id uuid default gen_random_uuid() primary key,
  title text, description text, requirements jsonb,
  created_at timestamp default now()
);
create table if not exists candidates (
  id uuid default gen_random_uuid() primary key,
  job_id uuid references jobs(id),
  name text, email text, phone text,
  resume_text text, parsed_data jsonb,
  match_score integer, match_explanation jsonb,
  status text default 'pending',
  created_at timestamp default now()
);
create table if not exists ai_cache (
  id uuid default gen_random_uuid() primary key,
  cache_key text unique not null,
  namespace text, result jsonb,
  created_at timestamp default now()
);
```

Start backend:
```bash
uvicorn main:app --reload --port 8000
```

### 3. Frontend setup
```bash
cd frontend
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173)
